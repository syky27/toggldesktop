# ADR-0001 — State management, FFI threading, and app conventions

**Status:** Accepted · **Date:** 2026-06-24 · Relates to issues FP-02, FP-22, FP-24.

## Context

The Flutter port (see `IMPLEMENTATION_PLAN.md`) drives the existing C++ core through the
`src/toggl_api.h` C ABI. Two cross-cutting decisions must be fixed before UI work begins:
state management, and how the core's push-callbacks reach Dart safely.

## Decision 1 — State management: **Riverpod**

We use `flutter_riverpod`.

- The FFI layer is exposed once as `coreServiceProvider`.
- Each `toggl_on_*` callback becomes a broadcast `Stream`; UI state is `StreamProvider`s /
  `NotifierProvider`s fed by those streams (see FP-24).
- Rationale: compile-safe DI, easy testing/mocking of `CoreService`, no `BuildContext` needed
  to reach state from the FFI isolate boundary.

## Decision 2 — FFI callback threading (the FP-22 crux)

The core invokes `TogglDisplay*` callbacks **from its own background threads**, and the view
structs (e.g. `TogglTimeEntryView*`) are **only valid for the duration of the synchronous call**
(the core frees/reuses them after the callback returns).

Dart FFI offers two callback flavors:
- `NativeCallable.isolateLocal` — runs synchronously on the *calling* thread, but that thread must
  already have an attached Dart isolate. The core's threads do not → unusable directly.
- `NativeCallable.listener` — safe to call from any native thread, but runs the Dart code *later*
  on the isolate event loop. By then the view pointer is dangling.

**Chosen approach: a thin C bridge shim (`app/native/bridge.c`).**
The shim registers itself with `toggl_on_*`, runs synchronously on the core's thread, **deep-copies**
each view struct into a flat, heap-owned C buffer, and forwards the *owned* buffer pointer to a
`NativeCallable.listener`. Dart reads the buffer on the event loop and `free()`s it. This is the
only approach that is both thread-safe and pointer-safe.

Until the shim lands, `CoreService` contains a **direct-`.listener` fallback** guarded behind a
flag, usable for callbacks that pass only scalars/owned strings (e.g. `on_login`, `on_error`,
`on_online_state`); struct-bearing callbacks (`on_time_entry_list`, `on_timer_state`) require the
shim and are marked TODO in code.

## Conventions

- Layout: `app/lib/src/{native,models,state,ui}/`.
- All C strings are UTF-8 (`char_t == char`); the core is compiled **without** `UNICODE` so Windows
  also uses the `char` ABI (avoids `wchar_t` marshalling). This is a build-flag requirement on FP-10.
- Bindings are generated by `ffigen` from `src/toggl_api.h` (config `app/ffigen.yaml`); the
  hand-written bindings in `toggl_bindings.dart` are a focused POC subset that ffigen will supersede.
- Errors surface via the `on_error` stream → a global snackbar/toast.
